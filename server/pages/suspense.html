“괜찮을지도” 프로젝트에서 관리자 페이지를 만들게 되었다.
요구사항이라면 회원, 지도, 핀에 대한 조회와 수정 및 삭제 기능이 전부인 간단한 프로젝트이기 때문에 메인 프로젝트에서 리팩토링 중인 부분을 개발 시작 단계에서부터 염두에 두면서 시작했다.
 
우선 React 18의 핵심 기능 중 하나인 Concurrency(동시성)와 관련있는 suspense를 적용해봤다.
여기서 동시성이란 독립적으로 실행될 수 있는 작업들을 여러 조각으로 나누어 번갈아 가면서 실행하는 것을 말한다.
브라우저의 예를 들자면 브라우저는 네트워크 통신, 사용자 이벤트 처리 등 여러가지 작업을 처리해야하는데 하나의 작업만 수행한다면 나쁜 사용자 경험을 줄 수 밖에 없다.
실제로는 이벤트 큐와 이벤트 루프를 통해서 동시성을 유지한다.
 
비슷하게 리액트도 렌더링 과정에서 일어나는 일들을 작은 단위로 나눠 번갈아가면서 작업을 처리해 동시성 모드를 지원한다.
조금 더 구체적으로 "뒷배경 작업(backburner)"을 통해 비동기적으로 작업을 처리하고, 사용자 인터랙션에 영향을 미치지 않으면서도 빠른 초기 렌더링을 제공한다.
 
Suspense는 이러한 비동기 작업을 보다 효율적으로 처리할 수 있게 도와준다.
동시성 작업의 지연을 관리하고 로딩 상태를 처리하는 데 사용된다.
추가적으로 개발자는 컴포넌트에서 사용할 데이터가 “정상적으로 로드되었을 때”의 UI만 고려하면 되고, 로딩 중이나 에러의 상태에서의 UI는 위임하면 된다. 더 선언적인 코드가 된다고 볼 수 있다.
 
아직 메인 프로젝트에서는 tanstack-query, swr과 같은 라이브러리 없이 fetch 메소드를 통해서만 데이터를 패칭하기 때문에 리액트에서 제공하는 suspense를 적용하지 못하는 상태이다.
간략히하면 아래와 같은 형태로 데이터 패칭 중이다.
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then((response) => response.json())
  .then((data) => console.log(data));
 
 
사용자 경험을 개선하기 위해 조금 더 고도화해서 데이터를 가지고 오는 중인 상태, 에러가 발생한 상태를 가지는 형태로 아래와 같이 고도화를 할 수 있다.
export function useFetchDataWithStatus(url: string) {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);
  const [isError, setIsError] = useState(false);

  const fetchData = async () => {
    try {
      const response = await fetch(url);

      if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);
      }

      const jsonData = await response.json();
      setData(jsonData);
    } catch (error) {
      setIsError(true);
    } finally {
      setIsLoading(false);
    }
  };

  return { data, isLoading, isError};
}
 
위와 같은 방식은 아직 단점이 많이 남았다.
useFetchDataWithStatus를 사용한 컴포넌트안에서 여전히 loading상태와 에러 상태에 대해서 꽤나 많은 분기를 가지고 있다. <Suspense>를 사용하기 위해 다음 두가지의 내용에 집중해서 useFetch를 구현했다.
  1. children에 속하는 컴포넌트가 예외 처리로 Promise를 던지게 되면(throw) fallback 프로퍼티로 전달받은 컴포넌트를 렌더링 한다
  1. 예외로 던져진 Promise가 완료 되면(fulfilled) children을 다시 렌더링 한다.
 
const useFetch = <T>(fetchFunction: (url: string) => Promise<T>, url: string): T | undefined => {
  const [promise, setPromise] = useState<Promise<void>>();
  const [status, setStatus] = useState<'pending' | 'fulfilled' | 'error'>('pending');
  const [result, setResult] = useState<T>();
  const [error, setError] = useState<Error>();

  useEffect(() => {
    setStatus('pending');
    setPromise(
      fetchFunction(url)
        .then((data) => {
          setStatus('fulfilled');
          setResult(data);
        })
        .catch((err) => {
          setStatus('error');
          setError(err);
        })
    );
  }, [url]);

  if (status === 'pending' && promise) {
    throw promise;
  }

  if (status === 'error') {
    throw error;
  }

  return result;
};

export default useFetch;
